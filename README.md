# 🚀 INNOVEX - Campus Innovation OS

**An AI-powered innovation accelerator for universities**

INNOVEX is a comprehensive campus innovation platform that connects students, mentors, and investors through AI-driven matchmaking, gamification, and ecosystem intelligence.

- **Deployment:** Vercel (with cron jobs)

## 📦 Installation

### Prerequisites
- Node.js 18+ and npm
- Supabase account
- Groq API key
- Hugging Face API token

### 1. Clone Repository
```bash
git clone <repository-url>
cd campus-connect
```

### 2. Install Dependencies
```bash
npm install
```

### 3. Set Up Environment Variables
Copy `.env.example` to `.env.local` and fill in your API keys:

```bash
cp .env.example .env.local
```

Required variables:
- `NEXT_PUBLIC_SUPABASE_URL` - Your Supabase project URL
- `NEXT_PUBLIC_SUPABASE_ANON_KEY` - Supabase anon public key
- `SUPABASE_SERVICE_ROLE_KEY` - Supabase service role key (admin access)
- `GROQ_API_KEY` - From https://console.groq.com/keys
- `HUGGINGFACE_API_KEY` - From https://huggingface.co/settings/tokens
- `CRON_SECRET` - Generate with `openssl rand -base64 32`

Optional variables:
- `GOOGLE_CALENDAR_CLIENT_ID` / `GOOGLE_CALENDAR_CLIENT_SECRET` - For meeting scheduler
- `LINKEDIN_CLIENT_ID` / `LINKEDIN_CLIENT_SECRET` - For LinkedIn OAuth sharing

### 4. Set Up Database
Run the SQL schema in your Supabase SQL Editor:

```bash
# Copy contents of database/schema.sql to Supabase SQL Editor
# This creates:
# - 11 tables (profiles, startups, matches, missions, badges, notifications, etc.)
# - pgvector extension
# - RLS policies
# - Seed data (9 predefined badges)
```

Enable pgvector extension (if not already enabled):
```sql
CREATE EXTENSION IF NOT EXISTS vector;
```

### 5. Run Development Server
```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000) to view the app.

## 🚀 Deployment (Vercel)

### 1. Connect Repository
1. Go to [Vercel Dashboard](https://vercel.com/dashboard)
2. Click "Add New Project"
3. Import your Git repository

### 2. Configure Environment Variables
Add all variables from `.env.local` to Vercel:
- Go to Project Settings → Environment Variables
- Add each variable for Production, Preview, and Development

### 3. Enable Cron Jobs
The `vercel.json` file configures 3 cron jobs:
- **Hourly KPI Refresh** - `/api/cron/refresh-kpis` (every hour)
- **Weekly Report** - `/api/cron/weekly-report` (Monday 9am)
- **Daily Missions/Streaks** - `/api/cron/daily` (daily 8am)

Cron jobs are automatically enabled on Vercel Pro plans.

### 4. Set Up Custom Domain (Optional)
- Go to Project Settings → Domains
- Add your custom domain
- Configure DNS according to Vercel instructions

### 5. Deploy
```bash
git push origin main
```

Vercel will automatically deploy on push.

## 📊 Database Schema

### Core Tables
- `profiles` - User profiles with embeddings (student, mentor, investor, admin)
- `startups` - Student startup projects with stage tracking
- `matches` - Student-mentor connections with compatibility scores
- `missions` - Daily personalized challenges for students
- `badges` - 9 predefined achievement badges with JSONB criteria
- `user_badges` - Junction table tracking earned badges
- `notifications` - In-app notification system (7 types)
- `activity_log` - Platform-wide activity tracking
- `investor_pipeline` - Deal flow Kanban board for investors
- `meetings` - Mentor-mentee meeting scheduler
- `kpi_cache` - Weekly ecosystem insights cache

### Row Level Security (RLS)
All tables have RLS policies enabling:
- Users can only view/edit their own data
- Public read access for badge definitions
- Admin bypass for analytics and monitoring

- **Reasoning:** Groq Llama 3.3 70B generates compatibility explanations

### AI-Generated Content
- **Daily Missions:** Llama 3.1 8B Instant (~3 personalized tasks per day)
- **Match Explanations:** Llama 3.3 70B (2-3 sentence reasoning)

- **Ecosystem Reports:** Llama 3.3 70B (weekly strategic insights)

## 🔐 Security Features

### Authentication
- Supabase Auth with email/password
- Row Level Security (RLS) policies on all tables
- Role-based access control (student, mentor, investor, admin)
- Protected Server Actions with session checks

### Rate Limiting
- **AI Endpoints:** 3-10 requests/minute (expensive operations)
- **Mutations:** 30 requests/10 seconds
- **Queries:** 100 requests/10 seconds
- In-memory rate limiting (use Redis/Upstash for production scale)

### API Security
- Environment variables for all secrets
- Cron endpoints protected with `CRON_SECRET`
- Server-side validation on all mutations
- No sensitive data in client-side code

## 🐛 Troubleshooting

### TypeScript Module Resolution Errors
If you see "Cannot find module" errors for existing components:
```bash
# Clear Next.js cache and rebuild
rm -rf .next
npm run build
```

### Supabase RLS Issues
If queries return empty even with valid data:
1. Check RLS policies in Supabase Dashboard
2. Ensure `service_role_key` is used for admin operations
3. Verify user authentication state

### AI API Rate Limits
- **Groq:** 30 requests/minute on free tier
- **Hugging Face:** 1000 requests/month on free tier
- Implement caching and rate limiting for production

### Vector Search Not Working
```sql
-- Verify pgvector extension is enabled
SELECT * FROM pg_extension WHERE extname = 'vector';

-- Check if embeddings exist
SELECT COUNT(*) FROM profiles WHERE embedding IS NOT NULL;
```

## 🔄 Cron Jobs

### Daily Cron (`/api/cron/daily`)
**Schedule:** Every day at 8:00 AM  
**Tasks:**
- Send mission reminders to all students
- Check and send streak warnings for users at risk
- Run badge checking for active users

**Manual Trigger:**
```bash
curl -H "Authorization: Bearer YOUR_CRON_SECRET" \
  https://your-app.vercel.app/api/cron/daily
```

### Hourly KPI Refresh (`/api/cron/refresh-kpis`)
**Schedule:** Every hour  
**Tasks:**
- Update admin KPI cache
- Refresh ecosystem charts

### Weekly Report (`/api/cron/weekly-report`)
**Schedule:** Every Monday at 9:00 AM  
**Tasks:**
- Generate AI ecosystem insights with Groq Llama 3.3 70B
- Cache report for admin dashboard

## 🎨 Customization

### Adding New Badge Types
1. Add badge to `database/schema.sql` seed data
2. Implement criteria checking in `lib/gamification/badges.ts` → `checkBadgeCriteria()`
3. Add icon mapping in `components/shared/BadgeShowcase.tsx` → `BADGE_ICONS`

### Adding New Notification Types
1. Add type to `NotificationType` enum in notification actions
2. Create specialized notification function (e.g., `notifyCustomEvent()`)
3. Add icon mapping in `components/notifications/NotificationList.tsx` → `NOTIFICATION_ICONS`

### Modifying AI Prompts
- **Daily Missions:** `lib/ai/missions.ts` → `generateDailyMissions()`
- **Match Reasoning:** `lib/ai/groq.ts` → `generateMatchReasoning()`
- **Ecosystem Reports:** `lib/ai/groq.ts` → `generateWeeklyEcosystemReport()`
