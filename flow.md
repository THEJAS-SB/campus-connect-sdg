### PART 1: The User Journey Flow (Step-by-Step)

This is how users move through the platform from day one to graduation and beyond.

#### 1. Onboarding & Identity Generation

- **Action:** A user (Student, Mentor, Investor, or Admin) signs up using their institutional email.
- **System Response:** Supabase Auth verifies the email and generates a unique, permanent **RS ID**.
- **Routing:** Based on their selected role, the Next.js Middleware routes them to their specific dashboard.

#### 2. The Student Loop (The "Engine" of the ecosystem)

- **Profile Setup:** The student inputs their skills, interests (e.g., SDGs), and startup ideas.
- **Daily Engagement:** The student logs in to complete "Daily Missions" (generated by AI).
- **Progression:** As they update their startup stage (e.g., "Idea" → "MVP"), their **Innovation Score** increases.
- **Reward:** This score increase triggers their **Dynamic Avatar** to evolve (e.g., changing from a "thinking" animation to a "running" animation).
- **Broadcasting:** Upon hitting a major milestone (e.g., "MVP Deployed"), the system prompts a one-click notification to post the achievement to **LinkedIn** and **WhatsApp**.

#### 3. The Matchmaking Loop (Connecting the dots)

- **The Trigger:** A student requests mentorship, or a mentor/investor logs in looking for projects.
- **The AI Action:** The platform's vector engine scans all active RS IDs.
- **The Connection:** The Mentor/Investor dashboard populates with "Suggested Matches." Each match includes a clear, human-readable **Reasoning Factor** (e.g., _"Matched due to 85% overlap in EdTech and SDG 4"_).
- **The Handshake:** The mentor clicks "Connect," opening a direct WhatsApp chat or scheduling a meeting.

#### 4. The Admin Loop (The Bird's-Eye View)

- **Continuous Ingestion:** As students update statuses and mentors accept meetings, the database is continuously updated.
- **Automated Reporting:** The Admin logs in and instantly sees live charts showing the ecosystem's health (e.g., "40% increase in AI startups this month").
- **Strategic Insights:** They read the weekly "AI Growth Prediction" report to decide where to allocate institutional funding or which workshops to host next.

### PART 2: The Data & AI Flow (Under the Hood)

When a student clicks "Save Project," here is the exact microsecond-by-microsecond technical flow:

1. **Client Action (Next.js):** The student fills out a form and hits "Submit."
2. **Server Action (Next.js):** The data is securely sent to a backend Next.js Server Action. No separate API needed.
3. **Embedding Generation (Hugging Face):** The text describing the student's project is sent to a fast, free embedding model (`gte-small`) to be converted into an array of numbers (a vector).
4. **Database Storage (Supabase):** The text and the new vector are saved into the `startups` table in your PostgreSQL database using the `pgvector` extension.
5. **Real-time Trigger (Supabase + dotLottie):** The database update fires a webhook. The frontend detects the change, increments the student's Innovation Score, and seamlessly triggers the Lottie avatar animation state machine to celebrate.
6. **Background Matchmaking (Groq LLM):** \* The vector database runs a quick math calculation (cosine similarity) to find the 3 mentors with the most similar vector profiles.

- The raw data of these 3 mentors is sent to **Groq (Llama 3.3 70B)**.
- Groq quickly reads the profiles and generates the "Explainable Match Reason" text.

7. **Cache Invalidation (Next.js):** The `updateTag()` function is called, instantly refreshing the Mentor's dashboard so the new student appears in their "Suggested Mentees" list without them having to refresh the page.

### The Final "Big Picture"

**Data Flows UP:** Students create raw data -> Mentors/Investors act on it -> Admins measure it.
**Value Flows DOWN:** Admins secure funding based on metrics -> Investors/Mentors guide the projects -> Students build better startups and get hired.
